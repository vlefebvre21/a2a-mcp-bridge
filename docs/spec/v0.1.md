# v0.1 Specification — a2a-mcp-bridge

## Scope

Minimal MCP server providing three tools:

1. `agent_send(target: str, message: str, metadata: dict | None = None) -> SendResult`
2. `agent_inbox(limit: int = 10, unread_only: bool = True) -> list[InboxMessage]`
3. `agent_list() -> list[AgentRecord]`

Backed by SQLite. No auth. Local-first. Transport: MCP over stdio (v0.1) — SSE/HTTP deferred to v0.2.

## Agent identity

On MCP client connect, the client MUST identify itself via an initialization handshake parameter `agent_id` (string, 1–64 chars, `^[a-z0-9][a-z0-9_-]{0,63}$`). If missing, the server rejects the session.

Agents are ephemeral by default — they appear in `agent_list()` only while connected. A future `register` tool (v0.2) will allow persistent registration.

## Data model

```sql
CREATE TABLE agents (
    id TEXT PRIMARY KEY,
    first_seen_at TEXT NOT NULL,
    last_seen_at TEXT NOT NULL,
    metadata TEXT  -- JSON blob, optional Agent Card fragment
);

CREATE TABLE messages (
    id TEXT PRIMARY KEY,              -- ULID
    sender_id TEXT NOT NULL,
    recipient_id TEXT NOT NULL,
    body TEXT NOT NULL,
    metadata TEXT,                    -- JSON blob
    created_at TEXT NOT NULL,
    read_at TEXT,                     -- NULL = unread
    FOREIGN KEY (sender_id) REFERENCES agents(id),
    FOREIGN KEY (recipient_id) REFERENCES agents(id)
);

CREATE INDEX idx_messages_recipient_unread
    ON messages(recipient_id, read_at);
```

## Tool signatures

### `agent_send`

```python
def agent_send(
    target: str,        # recipient agent_id
    message: str,       # body, UTF-8, max 64 KiB
    metadata: dict[str, Any] | None = None,  # optional, JSON-serializable, max 4 KiB serialized
) -> SendResult
```

Result:
```json
{
  "message_id": "01HXYZ...",
  "sent_at": "2026-04-21T14:30:00Z",
  "recipient": "agent-b"
}
```

Errors:
- `TARGET_UNKNOWN` — target agent has never connected
- `TARGET_SELF` — sending to self is disallowed in v0.1
- `MESSAGE_TOO_LARGE`
- `METADATA_TOO_LARGE`

### `agent_inbox`

```python
def agent_inbox(
    limit: int = 10,           # 1..100
    unread_only: bool = True,
) -> list[InboxMessage]
```

Returns messages addressed to the calling agent (by its `agent_id`). When `unread_only=True`, only unread messages are returned AND marked as read atomically (SELECT + UPDATE in a single transaction).

InboxMessage:
```json
{
  "message_id": "01HXYZ...",
  "sender": "agent-a",
  "body": "hello",
  "metadata": {},
  "sent_at": "2026-04-21T14:30:00Z",
  "read_at": "2026-04-21T14:30:05Z"
}
```

### `agent_list`

```python
def agent_list() -> list[AgentRecord]
```

Returns agents seen in the last 7 days (configurable via `--active-window-days`).

AgentRecord:
```json
{
  "agent_id": "agent-a",
  "first_seen_at": "2026-04-20T10:00:00Z",
  "last_seen_at": "2026-04-21T14:30:00Z",
  "online": true,
  "metadata": {}
}
```

`online` = has an active MCP session with the bus right now.

## CLI

```bash
a2a-mcp-bridge serve [--db PATH] [--active-window-days N]
a2a-mcp-bridge init [--db PATH]             # create schema
a2a-mcp-bridge agents list [--db PATH]      # admin: list known agents
a2a-mcp-bridge messages tail [--db PATH]    # admin: stream latest messages
```

## Non-goals for v0.1

- ❌ Authentication / authorization
- ❌ Encryption at rest or in transit (relies on MCP transport security)
- ❌ Multi-host deployment (single SQLite file = single host)
- ❌ Message TTL / retention policy (manual cleanup only)
- ❌ Agent Cards (A2A full spec)
- ❌ Reply threading
- ❌ Streaming / push notifications (pull-only via `agent_inbox`)
- ❌ HTTP / SSE transport

## Testing strategy

- Unit tests for each repository method (CRUD on agents + messages)
- Integration tests via an in-process MCP client → MCP server → SQLite
- Property tests on message sizes + metadata validation
- Target: ≥90% line coverage on `src/a2a_mcp_bridge/` excluding `cli.py`
